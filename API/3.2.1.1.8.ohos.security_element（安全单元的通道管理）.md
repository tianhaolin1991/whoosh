# ohos.security_element（安全单元的通道管理）

本模块主要用于操作及管理安全单元（SecureElement，简称SE），电子设备上可能存在的安全单元有eSE(Embedded SE)和SIM卡。文档中出现的SE服务为SEService实例，参见 SEServer 。

```cangjie
import kit.ConnectivityKit.*
```
## Functions

### public func createService(): SEService

```cangjie
public func createService(): SEService
```
功能： 建立一个可用于连接到系统中所有可用SE的新连接（服务）。

**Returns:**

| type | description |
| --- | --- |
| SEService | 可用于连接到系统中所有可用SE的新连接（服务）。 |

**Exceptions:**

- **BusinessException**: BusinessException：对应错误码的详细介绍请参见 通用错误码 。

| code | description |
| --- | --- |
| 801 | Capability not supported. |



## Classes

### Channel

class Channel
```cangjie
public class Channel {}
```
功能： Channel的实例表示在某个Session实例上创建通道，可能为基础通道或逻辑通道。通过 Session.openBasicChannel 或 Session.openLogicalChannel 获取Channel实例。


### Reader

class Reader
```cangjie
public class Reader {}
```
功能： Reader的实例表示该设备支持的SE，如果支持eSE和SIM，则返回两个实例。通过 SEService.getReaders 获取Reader实例。


### SEService

class SEService
```cangjie
public class SEService {}
```
功能： SEService表示可用于连接到系统中所有可用SE的连接（服务），通过 createService 获取SEService实例。


### Session

class Session
```cangjie
public class Session {}
```
功能： Session的实例表示在某个SE Reader实例上创建连接会话。通过 Reader.openSession 获取Session实例。



## Enums

### ServiceState

enum ServiceState
```cangjie
public enum ServiceState <: Equatable<ServiceState> & ToString { | DISCONNECTED | CONNECTED | ... }
```
功能： 定义不同的SE服务状态值。

#### CONNECTED

```cangjie
CONNECTED
```
功能： SE服务状态已连接。

#### DISCONNECTED

```cangjie
DISCONNECTED
```
功能： SE服务状态已断开。
